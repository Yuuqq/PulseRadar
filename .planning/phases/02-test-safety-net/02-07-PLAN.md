<objective>
Lower the coverage fail-under gate from 80% to 28% as a regression ratchet.

Purpose: The verification report found that actual coverage is ~28% (13,171 statements), far below the aspirational 80% gate. This causes `pytest` to exit non-zero on every run, breaking the phase goal of "a developer can run one command" for a clean pass. The developer decided to set the gate to 28% as a floor — coverage can only go UP from here. Phase 3 tests will naturally push coverage higher. This preserves the gate mechanism (TEST-01) while making it actionable at the current coverage level (COV-05).

Output: Updated `pyproject.toml` with `--cov-fail-under=28` and a ratchet strategy comment.
</objective>
