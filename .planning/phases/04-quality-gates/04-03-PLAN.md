  <name>Task 1: Create .pre-commit-config.yaml</name>
  <files>.pre-commit-config.yaml</files>
  <action>
Create .pre-commit-config.yaml in the repository root with the following configuration:

```yaml
repos:
  - repo: https://github.com/astral-sh/ruff-pre-commit
    rev: v0.8.4
    hooks:
      - id: ruff
        args: [--fix]
      - id: ruff-format
```

This configuration:
- Uses the official ruff-pre-commit repository
- Pins to v0.8.4 (compatible with ruff 0.14.13 installed globally)
- Includes two hooks: ruff (linting with auto-fix) and ruff-format (formatting)
- The --fix arg allows ruff to automatically fix issues when possible

Use the exact configuration from RESEARCH.md "Pre-commit Configuration with Ruff" section.

  <action>
Install pre-commit hooks and verify performance:

1. Run `pre-commit install` to install hooks into .git/hooks/
2. Run `pre-commit run --all-files` to test hooks on the entire codebase (one-time verification)
3. Measure execution time - should complete in <10 seconds (RESEARCH.md estimates <1 second for 119 files)
4. Create a test commit to verify hooks work:
   - Stage a small change (e.g., add a comment to a Python file)
   - Run `git commit -m "test: verify pre-commit hooks"`
   - Verify hooks run automatically and pass
   - Unstage the test commit with `git reset HEAD~1`

If hooks take >10 seconds, investigate and optimize. RESEARCH.md indicates Ruff should be very fast (<1 second) for this codebase size.

IMPORTANT: Do not commit the test change - this is just to verify hooks work. Use `git reset HEAD~1` to undo the test commit.
